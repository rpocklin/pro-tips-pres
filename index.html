  • Overview

  • Every codebase has code reviewed regularly
  • PRs are the best process mature teams tend to follow
  • Time spent in PR feedback can slow down productivity and momentum.
  • So how can you improve it?

  • The Goals

  • Validate that the changes in code match the goals of the feature in terms of the following attributes:

  • naming conventions
  • code quality
  • testability
  • maintainability
  • security and performance

    • The Tips - Typescript

  • Prefer => to get properties
  • Prefer map object selector to switch statements
  • Prefer map or reduce to forEach and push
  • Don't forEach - use map, reduce, filter, find
  • Don't forEach - use includes, some, every

    • The Tips - Typescript (cont ...)

  • Don't abbreviate! eg. cNo, custNo vs. customerNumber
  • Don't use native Date() instead use:
  • moment / dayjs / date-fns

    • The Tips - Typescript (cont ...)

  • Cyclomatic complexity hurts the head:
  • indenting is the give-away.
  • Returning a boolean? Name isXXX or hasXXX
  • Embrace truthy and falsy
  • if (x.length > 0) is equivalent to if (x.length)

    • The Tips - Angular

  • One module per component. Always.
  • Testing observables (no async - use jasmine done())
  • Don't put logic into the HTML (only a simple ng-if)
  • Don't use ? in HTML:
  • it makes code fragile (eg. someVar?.value)
